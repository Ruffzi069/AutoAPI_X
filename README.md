# AutoAPI-X - Connected Vehicle API Security Simulation Platform

## Phase 1: Backend Foundation

AutoAPI-X is an educational and research-oriented automotive cybersecurity platform designed to demonstrate how modern connected vehicles rely on cloud APIs, telematics infrastructure, infotainment systems, and CAN networks.

### ⚠️ Important Notice

**AutoAPI-X is NOT:**
- A vehicle simulator
- A racing simulator
- A real-time driving simulation

**AutoAPI-X IS:**
- An educational cybersecurity platform
- A connected vehicle API demonstration tool
- A research platform for automotive security
- A CAN bus traffic simulation system

## Architecture Overview

Every vehicle action follows this flow:

```
User Action → API Request → Backend Service → CAN Frame Generation → 
Vehicle State Update → Log Creation → Real-Time Event Broadcast
```

## Installation

### Prerequisites

1. **Python 3.8 or higher**
2. **Linux system** (for SocketCAN support)
3. **can-utils** (for candump)

### Step 1: Install System Dependencies

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install python3 python3-pip can-utils

# Fedora/RHEL
sudo dnf install python3 python3-pip can-utils
```

### Step 2: Setup Virtual CAN Interface

```bash
# Load vcan kernel module
sudo modprobe vcan

# Create virtual CAN interface
sudo ip link add dev vcan0 type vcan

# Bring interface up
sudo ip link set up vcan0

# Verify interface is up
ip link show vcan0
```

### Step 3: Install Python Dependencies

```bash
cd backend
pip install -r requirements.txt
```

### Step 4: Run the Application

```bash
python app.py
```

The server will start on `http://localhost:5000`

## CAN Interface Validation

### Monitor CAN Traffic

In a separate terminal, run:

```bash
candump vcan0
```

This will display all CAN frames generated by AutoAPI-X in real-time.

### Expected Output

When you trigger vehicle actions via API, you should see CAN frames like:

```
vcan0  321   [8]  01 00 00 00 00 00 00 00    # Door lock
vcan0  321   [8]  02 00 00 00 00 00 00 00    # Door unlock
vcan0  323   [8]  03 00 00 00 00 00 00 00    # Boot open
vcan0  324   [8]  05 00 00 00 00 00 00 00    # Engine start
```

| ECU | CAN ID | Function |
|-----|--------|----------|
| Door ECU | 0x321 | Door lock/unlock |
| Horn ECU | 0x322 | Horn activation |

| Ignition ECU | 0x324 | Engine start/stop |

## Troubleshooting

### CAN Interface Not Found

If you see warnings about vcan0 not being available:

```bash
# Reload vcan module
sudo modprobe -r vcan
sudo modprobe vcan

# Recreate interface
sudo ip link add dev vcan0 type vcan
sudo ip link set up vcan0
```

### Permission Denied

If you get permission errors:

```bash
# Add user to dialout group (may require logout/login)
sudo usermod -a -G dialout $USER
```

### Port Already in Use

If port 5000 is already in use, modify `app.py`:

```python
socketio.run(app, debug=True, host='0.0.0.0', port=5001)
```
